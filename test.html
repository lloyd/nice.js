how to be nice(.js)

javascript is getting awesomer. you can now do cpu intensive shit like crypto in javascript. a problem that arises by doing compute intensive work is that you lock don't let the rest of your process (like, the browser) have any of the processor to do stuff like render user feedback. Some browsers even tell the user your code is broke if you run off and compute stuff for more than 5 or 10 seconds at a time. "web workers" let you run computationally expensive stuff in the background, but they aren't usable everywhere

nice.js is a teensy tinsy micro framework that lets you adjust instrument your code to be nicer, giving some processor time to the rest of the browser.
